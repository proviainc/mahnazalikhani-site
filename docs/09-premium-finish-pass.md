# Stage 09 - Premium Finish Pass

## Status

No built UI exists yet. This document defines the premium finish pass to run before implementation sign-off.

## Finish Priorities

1. Hero first impression.
2. Header balance.
3. Proof strip clarity.
4. CTA hierarchy.
5. Mobile first viewport.
6. Experience card specificity.
7. Footer composure.

## Required Finish Moves

During implementation review:

- Tighten section spacing if the page feels bloated.
- Relax spacing if sections feel cramped.
- Remove sections that repeat "structure" without new proof or action.
- Improve heading hierarchy where everything feels equal.
- Reduce visual weight of ProVia Events/EventAgent links if they compete with Calendly.
- Ensure cards feel handcrafted, not starter-template default.
- Remove any generic event-planner copy.
- Confirm no decorative motion is distracting from reading or trust.

## Pre-Signoff Questions

- Does the site feel like a senior personal brand?
- Does the homepage feel composed within five seconds?
- Is Calendly clearly dominant?
- Does proof appear early enough?
- Does mobile feel designed, not compressed?
- Are ProVia Events and EventAgent clearly contextual?
- Does anything look like a generic wedding/event template?

## Launch Verdict Rule

Do not treat implementation as complete if:

- Homepage average visual score is below `8.5`.
- Mobile quality is below `8.0`.
- Trust or CTA clarity is below `8.0`.
- The site feels technically correct but visually generic.
