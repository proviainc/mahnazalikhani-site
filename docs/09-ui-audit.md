# Stage 09 - UI Audit Checklist

## Status

No UI has been implemented yet. This checklist is for the implementation review.

## Required Widths

Inspect at:

- `1440`
- `1280`
- `1024`
- `768`
- `430`
- `390`
- `375`

## Header

Check:

- Name/wordmark size.
- Navigation spacing.
- Book Consultation CTA visibility.
- Mobile menu availability.
- No mixed tablet state where desktop CTA and mobile menu conflict.

## Hero

Check:

- H1 hierarchy.
- Calendly CTA visible early.
- LinkedIn secondary.
- Portrait/proof composition.
- Mobile first viewport clarity.

## Proof

Check:

- Metrics readable.
- No counter animation.
- No unapproved proof.
- Clean mobile stacking.

## Cards

Check:

- Consistent padding.
- Specific labels.
- No generic service-card feeling.
- No horizontal overflow.

## Footer

Check:

- Core links present.
- Trust links present.
- Footer does not become a link dump.
- Mobile stacking is readable.

## Accessibility

Check:

- Keyboard navigation.
- Visible focus states.
- CTA contrast.
- Body text contrast.
- Reduced-motion behavior.
- Tap target size.

## Link Checks

Verify:

- Calendly.
- LinkedIn.
- ProVia Events.
- EventAgent.
- Internal routes.
