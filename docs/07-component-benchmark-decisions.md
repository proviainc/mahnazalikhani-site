# Stage 07 - Component Benchmark Decisions

## Navbar

Benchmark:

- Calm, compact, balanced, and conversion-aware.

Must have:

- Disciplined name/wordmark sizing.
- Clear nav links.
- Primary CTA visible.
- Simple mobile menu.

Reject if:

- Logo/name dominates.
- CTA feels oversized.
- Mobile menu feels cramped or jumps.

## Hero

Benchmark:

- The visitor should understand Mahnaz's role, credibility direction, and next action within the first viewport.

Must have:

- Strong H1.
- Short support copy.
- CTA pair.
- Portrait/proof area.

Reject if:

- Headline is vague.
- Portrait pushes CTA too far down on mobile.
- Decorative imagery competes with the message.

## Proof Strip

Benchmark:

- Verified metrics should feel like quiet authority.

Must have:

- Clear numbers.
- Short labels.
- Consistent rhythm.

Reject if:

- Looks like a flashy counter section.
- Uses unapproved proof.
- Becomes too dense.

## Audience Cards

Benchmark:

- Help visitors self-identify quickly.

Must have:

- Four cards maximum.
- Specific audience labels.
- Operational concern per card.

Reject if:

- Cards read like generic services.
- All cards say the same thing.

## Selected Experience Cards

Benchmark:

- Prove experience without needing media-heavy case studies.

Must have:

- Event name/type.
- Scale where verified.
- Mahnaz's role or responsibility.

Reject if:

- Overstates private details.
- Reads like a long resume.
- Uses unapproved images.

## CTA Bands

Benchmark:

- Give visitors a calm, clear next step after trust is earned.

Must have:

- One primary CTA.
- Concise explanation.
- Optional LinkedIn secondary link.

Reject if:

- CTA band is too large.
- It introduces new actions.
- It repeats without adding context.

## Footer

Benchmark:

- Composed, complete, and easy to scan.

Must have:

- Core navigation.
- Calendly link.
- LinkedIn, ProVia Events, EventAgent context links.

Reject if:

- Link dump.
- Tiny unreadable text.
- External links compete visually with booking.
