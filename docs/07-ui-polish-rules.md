# Stage 07 - UI Polish Rules

## Header Polish

- Keep logo/name visually smaller than the hero headline.
- Header CTA should feel aligned with nav, not like a banner.
- Sticky header shadow should be subtle.
- Active states should be simple: color, underline, or background shift.

## Button Polish

- Primary buttons use copper background and warm ivory or white text.
- Secondary buttons use transparent or light surface with espresso text.
- Radius should be pill or softly rounded, consistently.
- Hover should darken or lift subtly, not jump.
- Icons are optional and should not clutter.

## Card Polish

- Use consistent padding.
- Use subtle border first; shadow only when needed.
- Avoid heavy drop shadows.
- Keep card headings aligned.
- Do not let cards become equal-weight filler.

## Proof Polish

- Numbers should be prominent but not oversized.
- Labels should be concise and readable.
- Proof should not look like a gamified stats dashboard.
- Do not animate counters.

## Image Polish

- Portrait crops should feel intentional.
- Use consistent radius and aspect ratio rules.
- Avoid awkward face crops on mobile.
- Do not mix many image treatments.

## Typography Polish

- Use display type for major editorial headings.
- Do not use display type for dense body sections.
- Keep body text color high contrast.
- Avoid all-caps paragraphs.
- Eyebrows can use small caps or letter spacing, but sparingly.

## Motion Polish

- No random one-off animations.
- Use shared variants.
- Hover states should be quiet.
- Respect reduced motion.

## Common Failure Modes To Avoid

- Oversized wordmark.
- Giant header button.
- Wedding-template blush/gold styling.
- Generic SaaS gradient hero.
- Cards with repetitive copy and no hierarchy.
- CTA repeated too often without new context.
- ProVia Events/EventAgent links competing with Calendly.
- Mobile hero where the user cannot see the CTA quickly.
