# Stage 07 - Layout Rules

## Global Layout

Use a consistent centered container:

- Standard max width: `1180px`.
- Wide max width: `1280px`.
- Text max width: `720px`.
- Narrow text max width: `620px`.

## Section Spacing

Desktop:

- Large sections: `128px`.
- Standard sections: `104px`.

Tablet:

- Standard sections: `80px`.

Mobile:

- Standard sections: `64px`.

Rules:

- Do not use random section padding.
- Keep proof close to claims.
- Use larger spacing only when the section changes topic or rhythm.

## Grid Rules

Use:

- Two-column hero when portrait is available.
- Four-card audience grid on desktop.
- Two-column grid on tablet.
- One-column stack on mobile.

Avoid:

- Uneven columns without intent.
- Dense four-column layouts on tablet.
- Horizontal overflow.

## Text Width

Rules:

- Long-form copy should not exceed `720px`.
- Editorial statements can be wider if line breaks remain readable.
- Proof labels should be short enough to scan.

## Hero Layout

Desktop:

- Text column around 52%.
- Image/proof column around 40%.
- Keep enough gap for composure.

Mobile:

- Stack text first.
- CTA before portrait if needed.
- Keep primary CTA visible without excessive scrolling.

## Content Rhythm

Recommended order within sections:

1. Eyebrow or label.
2. Heading.
3. Short explanatory copy.
4. Proof/detail grid.
5. CTA only if earned.

## Footer Layout

Desktop:

- 3 to 4 columns maximum.

Mobile:

- Stack with clear link groups.
- Keep booking CTA visible but not dominant over all footer content.
