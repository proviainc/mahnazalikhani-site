# Stage 08 - Editorial Publishing Model

## Launch Publishing Model

Mode:

- Code-managed pages only.

Owner:

- ProVia/Cursor-managed updates through repository changes and preview review.

Publishing flow:

1. Edit code/content.
2. Generate preview.
3. Review preview.
4. Approve merge.
5. Deploy through production branch.

## Insights / Blog Decision

No Insights or Blog section at launch.

Reason:

- The primary launch objective is personal credibility and consultation booking.
- No publishing cadence or non-technical editor requirement is confirmed.
- Empty or thin Insights pages would weaken quality.

## Future Publishing Model

If Insights is approved later:

- Admin path: Sanity Studio for regular non-technical publishing.
- Strategic path: Cursor-assisted article packages for SEO-sensitive thought leadership.
- Public routes: `/insights/` and `/insights/[slug]/`.
- Content model: post, category, optional tag.

## Future Article QA Requirements

Every future article should include:

- Search intent.
- Clear audience.
- SEO title and meta description.
- Clean slug.
- Excerpt.
- Internal links.
- Strong CTA.
- Factual consistency.
- No unsupported claims.
- Image and alt text.
- Answer-ready structure.
