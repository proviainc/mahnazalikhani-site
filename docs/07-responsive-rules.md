# Stage 07 - Responsive Rules

## Breakpoints

- Mobile: `0px` to `767px`.
- Tablet: `768px` to `1023px`.
- Desktop: `1024px` and above.
- Wide: `1280px` and above.

## Header

Desktop:

- Height: `76px`.
- Name/wordmark max width: `180px`.
- CTA visible in header.

Tablet:

- Keep nav if spacing remains clean.
- Collapse to menu if links crowd CTA.

Mobile:

- Height: `68px`.
- Name/wordmark max width: `150px`.
- Use simple menu.
- Include Book Consultation and LinkedIn in the mobile menu.

## Hero

Desktop:

- Two-column layout allowed.
- Proof can sit below copy or in adjacent card.

Tablet:

- Two-column only if portrait and text remain balanced.
- Otherwise stack.

Mobile:

- Stack content.
- H1 max should feel like `38px`.
- CTA tap targets at least `44px`.
- Avoid pushing proof too far below the fold.

## Cards

Desktop:

- Audience cards: 4 columns.
- Experience cards: 2 or 3 columns depending on content length.

Tablet:

- 2 columns.

Mobile:

- 1 column.
- No carousels by default.
- No horizontal scrolling card rows.

## Typography

Mobile:

- H1 around `38px`.
- H2 around `30px`.
- Body `16px`.
- Keep line lengths comfortable.

Desktop:

- H1 around `64px` max.
- H2 around `40px`.

## Spacing

Mobile:

- Section padding around `64px`.
- Card padding around `22px`.
- Grid gaps around `18px`.

Tablet:

- Section padding around `80px`.

Desktop:

- Standard section padding around `104px`.
- Large section padding around `128px`.

## Overflow Rules

- No layout should create horizontal scrolling.
- Images must use safe max-width behavior.
- Long URLs or external labels must wrap safely.
- Proof rows must stack before they overflow.

## Footer

Desktop:

- Use clean columns.

Mobile:

- Stack sections with clear headings.
- Keep external trust links readable.
- Avoid tiny legal/footer text.
