# Stage 07 - Visual Direction & Design System

## Desired Brand Perception

The site should feel:

- Senior
- Calm
- Editorial
- Warm
- Structured
- Premium through restraint

It should not feel like a wedding-planner template, a corporate event agency, a SaaS landing page, or a decorative resume site.

## Chosen Visual Direction

Direction:

- Editorial Event Strategist

This direction uses refined typography, warm neutrals, restrained contrast, structured layouts, and proof-led composition to make Mahnaz feel credible, personal, and senior.

The aesthetic should suggest:

- A composed professional people trust with a high-stakes room.
- Hospitality warmth without ornate luxury cliches.
- Operational structure without software-product coldness.
- Personal credibility without influencer-style branding.

## Design Principles

1. Structure is the premium signal.
   - Grids, spacing, alignment, and proof rhythm should feel disciplined.

2. Warmth must stay human.
   - Use portraiture, soft neutrals, and measured copy spacing to avoid a cold consulting feel.

3. Proof should carry visual weight.
   - Metrics and selected experience should be designed as credentials, not decorative badges.

4. Related brands stay contextual.
   - ProVia Events and EventAgent should appear as refined trust cues, not hero-level visual identities.

5. Mobile must feel composed.
   - The first viewport needs clear hierarchy, not compressed desktop leftovers.

## Color Palette And Usage

The palette should be warm, editorial, and composed.

Primary surfaces:

- Warm ivory for the base.
- Soft sand for alternate sections.
- Deep espresso for text and high-contrast areas.

Accent:

- Muted copper for CTA emphasis and small details.
- Deep teal as a secondary trust/operations accent.

Usage:

- Use warm ivory for most pages.
- Use deep espresso for footer and occasional high-authority blocks.
- Use copper sparingly for primary CTA, rules, labels, and small accents.
- Use teal sparingly for EventAgent/systems context or secondary highlights.
- Avoid bright gold-heavy luxury styling.

## Typography And Scale

Recommended type direction:

- Display/serif: editorial and senior.
- Body/sans: readable, modern, and calm.

Suggested families:

- Display: `Cormorant Garamond`, `Playfair Display`, or `Georgia` fallback.
- Body: `Inter`, `Lato`, or system sans fallback.

Hierarchy:

- H1 should feel editorial and confident, not giant for its own sake.
- Body copy should stay readable with generous line height.
- Proof labels should be compact, controlled, and highly legible.

## Grid And Spacing

Layout should use:

- Max content width around `1180px`.
- Text width around `720px`.
- Large desktop section spacing around `112px`.
- Mobile section spacing around `64px`.
- Card grids that collapse cleanly to one column.

Spacing should communicate control:

- Do not crowd proof.
- Do not create oversized dead zones.
- Keep CTA groups tight and intentional.

## Layout Rules And Breakpoint Behavior

Desktop:

- Header height: about `76px`.
- Hero can use a two-column composition when portrait is available.
- Proof strip should sit close enough to hero to reinforce the first claim.
- Text-heavy sections should use constrained width.

Tablet:

- Reduce columns before content feels cramped.
- Keep portrait and proof blocks balanced.

Mobile:

- Hero should stack copy first, then image/proof.
- CTA buttons should be full-width or near full-width only when it improves tap clarity.
- Avoid horizontal scroll, carousels, and oversized image crops.

## Key Component Styles

### Navbar

- Calm, compact, and stable.
- Logo/name treatment should not overpower nav.
- Primary CTA should be visible but not oversized.

### Hero

- Strong editorial headline.
- One clear CTA pair.
- Portrait area should feel composed and professional.
- Proof should be visible early.

### Proof Strip

- Use restrained metric cards or an aligned proof row.
- Keep numbers prominent and labels clear.
- Avoid flashy counters.

### Audience Cards

- Use four cards maximum.
- Each card should name a real audience and operational concern.
- Cards should feel like clarity tools, not generic service tiles.

### Selected Experience

- Use privacy-safe cards or timeline blocks.
- Emphasize role, event type, scale, and operational responsibilities.

### CTA Bands

- Compact, confident, and low-noise.
- Reinforce what the consultation covers.

## Component Benchmark Decisions

The most important components for first impression and conversion:

- Navbar.
- Hero.
- Proof strip.
- Audience cards.
- Selected experience cards.
- CTA bands.
- Footer.

Each should prioritize proportion, hierarchy, and trust over decoration.

## Page Composition Guidance

Home:

- Editorial hero with early proof.
- Alternating calm surfaces.
- Strong proof rhythm.
- Contained ProVia/EventAgent context.

About:

- More narrative and portrait-led.
- Use career milestones with structured chronology.

Experience:

- Proof-forward.
- Avoid gallery dependence until media is approved.

Event Strategy:

- Offer clarity and trust.
- More restrained and conversion-focused than decorative.

## Image And Iconography Direction

Portrait:

- Professional, warm, calm, direct.
- Natural light or premium indoor setting.
- Avoid event-party stock imagery.

Event imagery:

- Use only approved images.
- Prefer atmosphere, coordination, room flow, and professional presence over decorative table shots.

Iconography:

- Minimal line icons if needed.
- Use sparingly.
- Avoid generic event icons, confetti, balloons, stars, or excessive calendar/checklist imagery.

## Motion Guidance

Motion verdict:

- Minimal to moderate, closer to minimal.

Motion should feel:

- Composed.
- Smooth.
- Quiet.
- Useful.

Use:

- Subtle hero reveal.
- Fade-up section entry.
- Small card hover lift.
- Gentle nav state change.
- CTA hover feedback.

Avoid:

- Parallax.
- Decorative loops.
- Flashy counters.
- Large slide-ins.
- Animation in dense reading sections.

## Motion System Rules

Approved patterns:

- `fadeUp`
- `staggerGroup`
- `hoverLift`
- `navCompress`

Timing:

- Enter: `280ms` to `360ms`.
- Hover: `140ms` to `180ms`.
- Stagger: `60ms` to `80ms`.

Reduced motion:

- Disable scroll reveals and stagger.
- Keep only instant or near-instant opacity/state changes.

## Accessibility Rules

- Body text contrast must meet WCAG AA.
- Primary CTA must remain legible on all surfaces.
- Focus states must be visible.
- Tap targets should be at least `44px`.
- Do not communicate meaning through color alone.
- Reduced-motion behavior is required.
- Avoid tiny low-contrast proof labels.

## UI Polish Rules

- Logo/name must never feel oversized.
- Header CTA should not be taller than the nav rhythm.
- Cards should use consistent padding and border treatment.
- Avoid heavy shadows.
- Use one border/shadow language across the site.
- Keep radius subtle.
- Align all section content to a consistent grid.
- Do not use decorative gradients unless they are extremely subtle and purposeful.

## Stage 07 Exit Check

- Visual direction supports the strategy.
- Core design principles are defined.
- Design system is reusable.
- Accessibility and motion are considered.
- Component and layout direction are clear enough for build planning.
- The system avoids generic category defaults.
- Navbar, spacing, buttons, cards, motion, and responsive behavior are explicit.

Stage 07 is complete enough to proceed to Stage 08.
