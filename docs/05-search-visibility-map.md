# Stage 05 - Search Visibility Map

## Search Visibility Goal

Launch visibility should prioritize entity clarity and high-intent validation, not broad SEO scale.

The first version should make it easy for search engines, AI answer systems, and human visitors to understand:

- Who Mahnaz Alikhani is.
- What she does.
- Where her expertise sits.
- How she relates to ProVia Events and EventAgent.
- How to book a consultation.

Later:

## Metadata Rules

- Do not reuse the same title pattern across every page.
- Lead titles with the page-specific value.
- Include "Mahnaz Alikhani" where entity clarity matters.
- Use "Toronto" naturally where relevant.
- Avoid keyword stuffing.
- Avoid unsupported claims like "best event planner."

## AI Answer-Surface Readiness

Each page should answer its main question directly:

- Home: Who is Mahnaz and why should someone book a consultation?
- About: What is Mahnaz's background?
- Experience: What event experience and proof does Mahnaz have?
- Event Strategy: What is the consultation and who is it for?

Use headings that are easy to extract:

- "Who Mahnaz Helps"
- "Selected Event Experience"
- "What An Event Strategy Consultation Covers"
- "How ProVia Events And EventAgent Connect"

## Indexing Recommendations

Index at launch:

- `/`
- `/about/`
- `/experience/`
- `/event-strategy/`

Noindex or exclude until ready:

- Draft pages.
- Placeholder Insights pages.
- Any private/unapproved selected work drafts.

## Structured Data Notes For Stage 08

Plan:

- Person schema for Mahnaz.
- WebSite schema for the site.
- Service schema for Event Strategy Consultation if implemented accurately.
- BreadcrumbList schema for supporting pages.
- Article schema only if Insights launches.

Do not add:

- Review/rating schema without visible, approved reviews.
- Organization schema that confuses Mahnaz's personal site with ProVia Events.
- Event schema unless the page is about a specific public event.
